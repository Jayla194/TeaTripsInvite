# Tea Trips User Testing Site

A lightweight, standalone microsite for Tea Trips user testing. It mirrors the Tea Trips visual brand and clearly guides testers to feedback while keeping the process casual and ethical.

## What This Is
This is a **public user‑testing landing page** for my final‑year Tea Trips project. It explains the test, links to the app, and routes testers to the feedback/bug forms. If you found this on GitHub, it is here to showcase my product/UX communication and branding work.

Highlights:
- Static HTML/CSS/JS (no build step)
- Designed for GitHub Pages hosting
- Emphasizes clear instructions and anonymous feedback

## Local Preview
Open `UserTesting/index.html` in your browser.

## Deploy To GitHub Pages
1. Push this folder to your GitHub repo.
2. In GitHub: `Settings` → `Pages`.
3. Source: `Deploy from a branch`.
4. Branch: `main` (or `master`), Folder: `/UserTesting`.
5. Save and wait for the Pages URL.

## Customize
- Update the app link in `UserTesting/index.html`.
- Update the Microsoft Forms links in `UserTesting/index.html`.
- Update the contact email in `UserTesting/index.html`.
- Replace `UserTesting/TeaTripsLogo.png` if you want a new logo.

## Why It Matters
This microsite demonstrates brand consistency, thoughtful UX communication, and a testing workflow aligned with university ethics requirements.
